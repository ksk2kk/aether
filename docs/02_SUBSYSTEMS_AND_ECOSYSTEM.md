# 2. 子系统构建与生态发行版指南

Aether Hypercore 仅提供物理原语。一个完整的操作系统发行版，由一系列互相协作的预置隔离域（存储、网络、图形合成）及生态兼容层组成。

## 2.1 存储矩阵：内存映射去重账本 (MMDL)
废弃传统操作系统的 VFS 和块设备抽象，Hyperion 实施基于内存映射的直接寻址拓扑。

*   **存储即内存**：NVMe 设备由底层存储域接管，数据读写被抽象为物理页映射。
*   **硬件写时复制与去重**：百个相同容器实例请求 libc 库，存储域计算 Hash 后，通过 `HypercallMmdlMapShared` 将单一物理页以**只读**映射至百个 EPT 中。任一域尝试写入时，硬件抛出 EPT Violation，由 Hypervisor 分配新页，实现零开销的硬件去重与隔离机制。

## 2.2 生态承载：Aether Fusion
通过增强型 POSIX 宏域无痛接入存量 Linux 桌面及服务端生态。不要求修改一行存量应用代码。

### 宏域 I/O 融合 (Macro-Realm I/O Fusion)
宏域内部抛弃原生硬件驱动，替换为 Aether 特制的“融合驱动”，将 I/O 路径桥接至微域：
*   **网络融合 (`eth-fusion`)**：将宏域内的 Socket 流经由内存语义总线 (MSB) 直接抛给硬件直通的独立网络微域，绕过宏域内核协议栈。
*   **存储融合 (`blk-fusion`)**：对接 MMDL 层，宏域数据库即可透明享有硬件去重与原子快照。
*   **图形融合 (`v-gfx`)**：宏域内应用渲染完成的 Framebuffer，通过 MSB 原子化地移交至隔离的合成器域，保障主系统在显示驱动崩溃时的绝对可用性。

## 2.3 融合渲染管线 (Fused Rendering Pipeline)
UI 与桌面环境的物理防线设计：
1.  **无头前端域**：在宏域内提供 X11/Wayland 兼容桩，收集应用的绘图指令。
2.  **合成器微域**：高可用微域，通过 `HypercallPageTransfer` 从无头前端接收 Framebuffer 的**物理页所有权**。
3.  **直通驱动域**：合成器将最终帧所有权通过 MSB 转交给持有闭源显卡驱动的隔离域，直接驱动硬件执行 DMA 读取。数据流动严格遵循零拷贝和权限降维原则。

## 2.4 发行版开发向导
开发者构建新一代操作系统的职责是将服务打包为微域并编排启动流。

1.  **服务分离**：开发独立的存储域（控制 NVMe）、网络栈域（控制 NIC）、Init 管理域。
2.  **启动声明**：Init 域作为第一个被 Hypercore 唤醒的系统，通过 Hypercall 解析配置，动态分配 EPT，拉起其他工作域。
3.  **跨域对象交互**：系统 CLI 或服务间通信不再使用字节流管道 (`|`)，而是在内存构建结构化对象后，通过 MSB 原子转移页的所有权。
